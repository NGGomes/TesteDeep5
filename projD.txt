# ─── COMO O AGENTE 1 PROCESSA ESTE DATASET ───────────────────────────────────
#
# O Agente 1 NÃO produz um card por janela temporal. Produz um card por
# CLUSTER de alertas, onde um cluster é definido pela combinação de:
#   • categoria CISO primária (primary_ciso) do alerta âncora
#   • janela de tempo deslizante (sliding window)
#
# Consequência directa: uma janela temporal com 16 alertas e 7 fases pode
# gerar MÚLTIPLOS cards — um por categoria CISO dominante detectada —
# todos com o score acumulado da janela no momento em que são emitidos.
#
# ─── MAPA DE FASES (correspondências exactas) ────────────────────────────────
#
#   (informationgathering, scanning|reconnaissance|osint)   → RECON            w=1
#   (authentication, login|brute force|credential stuffing) → INITIAL_ACCESS   w=3
#   (execution, script|command|powershell|malware)          → EXECUTION        w=4
#   (lateral_movement, rdp|wmi)                             → LATERAL_MOVEMENT w=5
#   (persistence, backdoor|registry)                        → PERSISTENCE      w=6
#   (exfiltration, data|data leak)                          → EXFILTRATION     w=7
#   (impact, ransomware|wiper|encryption|system)            → IMPACT           w=8
#
#   ATENÇÃO: Categorias fora deste mapa (ex: availability/ddos, other/compliance,
#   malicioscode/cryptominer) NÃO produzem fase de kill chain pelo caminho
#   principal. São processadas pelo mapeamento secundário via primary_ciso
#   (keyword matching em Title/Tags/Description), gerando cards com a fase
#   da categoria CISO mais próxima mapeada.
#
# ─── PHASE_TO_CISO_CATEGORY ──────────────────────────────────────────────────
#
#   RECON          → CISO-15
#   INITIAL_ACCESS → CISO-4
#   EXECUTION      → CISO-14
#   PERSISTENCE    → CISO-5 (APT)
#   LATERAL_MOVEMENT → CISO-5 (APT)
#   EXFILTRATION   → CISO-3
#   IMPACT         → CISO-1
#
#   Categorias CISO 2,6,7,8,9,10,11,12,13,16 são atingidas exclusivamente via
#   primary_ciso (keyword matching). Quando activadas, o card usa a fase de
#   kill chain do alerta que activou o keyword match.
#
# ─── SCORE → ESTADO CISO ─────────────────────────────────────────────────────
#
#   0          → RUIDO_OPERACIONAL   (Operational Noise)
#   1–2        → RUIDO_OPERACIONAL   (apenas RECON; Operational Noise)
#   3–5        → ATIVIDADE_SUSPEITA  (Suspicious Activity)
#   6–9        → INCIDENTE_SEGURANCA (Security Incident)
#   10–14      → INCIDENTE_GRAVE     (Serious Incident)
#   ≥ 15       → RANSOMWARE_PROVAVEL (Critical Combined Attack)
#
# ═══════════════════════════════════════════════════════════════════════════════
